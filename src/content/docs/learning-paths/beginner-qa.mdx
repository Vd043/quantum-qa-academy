# Beginner QA Path

**Who this is for:**  
New QA engineers, career switchers, or developers who want to understand testing deeply.

**Prerequisites:**  
Basic computer literacy, some familiarity with web or mobile apps.

## Phase 1 – Foundations of QA

- Read: [QA SDLC Overview](/qa-sdlc/overview)  
- Read: [Test Strategy & Plans](/qa-sdlc/test-strategy)

**Outcome:** You understand the full QA lifecycle and key deliverables.

## Phase 2 – Designing Good Tests

- Read: [Test Design Techniques](/qa-sdlc/test-design-techniques)
- Exercise: Take a simple app and write test cases using at least three techniques.

## Phase 3 – Execution & Reporting

- Read: [Test Execution & Reporting](/qa-sdlc/test-reporting)
- Practice: Execute a small test suite and write a concise test report.

## Done when

You can:

- Explain the QA SDLC to a teammate.
- Write a basic test plan and a small set of well-designed test cases.
- Run tests and communicate status clearly.
